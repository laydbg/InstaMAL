The environment is a managed client network containing two Windows workstations, two Windows servers, and a domain controller, all connected on a shared internal network. The network has outbound connectivity to an external network representing internet or service provider infrastructure. Domain authentication is managed centrally by the domain controller, which holds a credential database containing hashed credentials for all domain accounts.

Each machine has a local account database storing hashed credentials for its local user. One workstation additionally holds credentials for its local user in a more accessible in-memory form. Each machine has a dedicated local user account operating at standard privilege. A domain administrator account holds elevated access to the workstations, the servers, and the domain controller, with its credentials stored in the domain controller's central database.

The structurally distinctive feature of this environment is a second high-privilege administrative account representing the managed service provider's administrative identity. This account holds elevated access to every machine in the environment without exception, including both workstations, both servers, and the domain controller. This universal administrative access reflects the trusted relationship between the service provider and the client network, and means that any party holding these credentials has an immediate privileged foothold on every asset in the environment simultaneously.
