The environment is a mixed-platform organisation network containing a Linux server, two Windows workstations, and a Windows domain controller, all connected on a shared internal network. The Linux server is accessible via SSH using a standard user account. A privilege escalation path exists on the Linux server through a vulnerable setuid binary that can be abused to gain elevated access. The Linux server holds two categories of sensitive credential material: a system password file containing both cleartext and hashed root credentials, and SSH key material stored in the user's home directory.

The user account on the Linux server shares credentials with a local account on one of the Windows workstations, forming a cross-platform credential bridge that links the Linux environment to the Windows environment. That Windows workstation holds a browser credential store readable by the local user account, and a keylog file containing captured domain administrator credentials. A keylogger vulnerability on the same workstation captures credentials during interactive user sessions, requiring a legitimate user to log in for the credential capture to occur.

A domain administrator account holds elevated access to both Windows workstations and the domain controller. Each of these three machines holds a sensitive user data store representing files of value to the organisation. The domain controller has no additional credential stores modeled beyond its role as a high-privilege access target. The second Windows workstation has no local credential stores and serves primarily as a lateral movement target reachable via the domain administrator account.
