The environment is a retail organisation's internal network containing a web server, two Windows workstations, a point-of-sale system, a file sharing server, and a domain controller, all connected on a shared internal network. The web server is the boundary asset of the environment: it is reachable from the internet and simultaneously connected to the internal network, making it both a public-facing service and a potential pivot point into the internal environment. All other assets are connected only to the internal network, with one workstation having outbound internet access.

Domain authentication is managed by the domain controller. A single domain administrator account holds elevated access to every system in the environment, including the web server, both workstations, the point-of-sale system, the file sharing server, and the domain controller. The web server additionally allows unauthenticated external access, representing its role as a publicly accessible application. Purchase request data flows through the web server, representing payment information submitted by external users.

One workstation has a local administrator account whose credentials are stored in accessible memory on that machine. A software vulnerability on the same workstation allows a low-privileged user to escalate to local administrator level. Domain administrator credential hashes are also stored on this workstation, making it a high-value target for credential harvesting. The point-of-sale system holds payment card track data as its primary sensitive asset and is accessible only from within the internal network.
