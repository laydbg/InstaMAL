The environment is an organisation's internal network containing an Exchange administration workstation, an Exchange email server, a SQL database server, and a domain controller, all connected on a shared internal network. The Exchange Server is the boundary asset of the environment: it is reachable from the internet and simultaneously connected to the internal network, serving as both a public-facing email service and an internal application accessible from within the organisation.

Two privileged accounts operate with overlapping but distinct access rights. The first is an Exchange administrator account that holds low-privilege access to the workstation and high-privilege access to the Exchange Server. This account's credentials are stored in a credential vault on the workstation, accessible to the local user session. The second is a higher-level administrator account with high-privilege access to both the Exchange Server and the SQL Server. This account's credentials are stored in accessible memory on the Exchange Server, making the Exchange Server a pivot point between the two administrative privilege tiers.

The workstation has outbound internet access. A legitimate user operates the workstation under the Exchange administrator account and is the target of social engineering for initial access. The SQL Server holds a database backup as its primary sensitive data asset and is accessible only through the higher-level administrator account. The domain controller is present on the internal network but holds no credential stores relevant to the attack path and serves primarily as a domain infrastructure component.
